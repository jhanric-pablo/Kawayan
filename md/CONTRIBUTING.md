# Contributing to Kawayan AI

[**⬅ Back: System Manual**](./SYSTEM_MANUAL.md) | [**Next: Testing Guide ➔**](./TESTING_GUIDE.md)

We welcome contributions to Kawayan AI! As a project focused on helping Philippine SMEs, we value code quality, local context, and user-centric features.

## 1. Code of Conduct
Please be respectful and professional in all interactions. Our goal is to create a supportive environment for all contributors.

## 2. Development Workflow
1.  **Fork the Repository**: Create your own copy of the project.
2.  **Create a Branch**: Use descriptive names like `feature/new-post-format` or `fix/wallet-sync`.
3.  **Implement Changes**: Follow the project's coding style (TypeScript, functional React, Tailwind).
4.  **Write Tests**: Ensure your changes are covered by basic or system tests.
5.  **Submit a Pull Request**: Provide a clear description of your changes and why they are needed.

## 3. Coding Standards
-   **TypeScript**: Use strict typing where possible.
-   **Components**: Prefer functional components and hooks.
-   **Styling**: Use Tailwind CSS utility classes.
-   **Documentation**: Update relevant `.md` files if you change the database schema, API, or system behavior.

## 4. Reporting Issues
Use the GitHub issue tracker to report bugs or suggest enhancements. Please include:
-   A descriptive title.
-   Steps to reproduce (for bugs).
-   Expected vs. actual behavior.
-   Screenshots or logs if applicable.

---

## 5. Local Context Guidelines
When writing AI prompts or UI copy, remember the primary audience: **Philippine SMEs**.
-   Use **Taglish** where appropriate for captions.
-   Keep the tone **warm, helpful, and community-oriented**.
-   Ensure pricing and currency references are in **PHP (₱)**.
